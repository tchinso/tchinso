## What you need to do now

딱 **3가지만** 확인하면 됩니다.

1. **이 저장소 이름이 정확히 `tchinso`인지 확인**
   - GitHub 프로필 README는 **`<your-username>/<your-username>`** 저장소여야 프로필에 표시됩니다.
   - 즉 지금 저장소가 `https://github.com/tchinso/tchinso` 여야 합니다.

2. **GitHub Actions를 켜기**
   - 저장소 페이지에서 **Settings → Actions → General** 로 이동합니다.
   - **Actions permissions** 는 기본값으로 두어도 보통 괜찮습니다.
   - 가장 중요한 건 아래쪽의 **Workflow permissions** 를 **Read and write permissions** 로 바꾸는 것입니다.
   - 이 설정이 있어야 workflow가 `README.md`를 자동 커밋할 수 있습니다.

3. **한 번 수동 실행하기**
   - **Actions** 탭으로 이동합니다.
   - **Update profile README** workflow를 엽니다.
   - **Run workflow** 버튼을 눌러 1회 실행합니다.
   - 실행이 끝나면 `README.md`가 최신 데이터로 자동 갱신됩니다.

## Do you need to issue an API key?

**아니요. 별도 API 키나 Personal Access Token(PAT)은 없어도 됩니다.**

이 워크플로는 GitHub가 기본 제공하는 **`GITHUB_TOKEN`** 을 사용합니다.
즉, 따로 토큰을 발급해서 Secrets에 넣지 않아도 됩니다.

필요한 건 사실상 이것뿐입니다.

- 저장소 이름이 `tchinso`
- Actions 활성화
- Workflow permissions를 **Read and write** 로 변경
- `Run workflow` 1회 실행

## If it does not update

아래를 순서대로 보면 됩니다.

- Actions 실행 로그에서 Python step이 실패했는지 확인
- **Workflow permissions** 가 `Read and write` 인지 다시 확인
- 저장소가 정말 프로필 저장소(`tchinso/tchinso`)인지 확인
- README가 바뀌었는데 프로필 반영이 늦으면 몇 분 정도 기다리기
